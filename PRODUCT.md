# Product

## Register

brand

## Users

Two audiences arrive cold from LinkedIn, GitHub, or referrals. First: prospective clients (product, engineering, and ops leads) evaluating AI-native product help. Second: frontier-lab hiring managers screening for staff-level AI product engineering and design engineering. Both decide in under a minute, on desktop or phone.

## Product Purpose

infolog.io is Brendan Lee's consultancy hub and portfolio gateway. Phase 1 is a one-screen placeholder: state the positioning, capture calls. Phase 2 is a bento portfolio gateway built on infolog.lib showing five projects (crow.pet, multiplex.chat, skillstud.io, monet.haus, infolog.lib). Success: a visitor books a call or remembers the page.

## Brand Personality

Editorial, precise, principal-led. A typeset broadsheet with one impossible element: a sky that knows the visitor's time. Big serif display over quiet mono labels. Confidence through restraint, not volume.

## Anti-references

- Generic AI-consultant SaaS pages: gradient buttons, hero metrics, identical feature-card grids.
- Yellow-cream "parchment" backgrounds and ash-gray ink (explicitly rejected by Brendan, 2026-06-10).
- Decorative dashboard chrome, fake telemetry, marketing buzzwords.
- Lead-gen friction: email-capture forms, demo gates.

## Design Principles

1. One signature visual trick per page; everything else stays simple (Spiral/Cora/Sparkle DNA).
2. Type carries the brand: Fraunces display, IBM Plex Mono labels, big sizes.
3. Color derives from the artwork: UI ink pulls from the live sky palette, contrast-clamped.
4. CTAs go straight to the calendar or mailto. No forms.
5. One static file per page: no build step, no dependencies beyond fonts.

## Accessibility & Inclusion

WCAG 2.1 AA. Derived ink clamps to 4.5:1 minimum on paper. Reduced motion receives a static, time-correct frame. No-WebGL and no-JS both fall back to plain CSS gradients.
